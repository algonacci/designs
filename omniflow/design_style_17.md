# Omniflow Design Style 17: Input Lab

**Cocok untuk:** Analytics platform, parser architecture, multi-source data, integrations.

## Creative Thesis

Setiap sumber data memiliki struktur sendiri. Platform analytics bekerja seperti laboratorium yang menerima sample berbeda, memilih instrumen yang tepat, lalu menghasilkan pembacaan yang dapat dibandingkan.

## Signature Device

**Sample Cartridge:** satu cartridge sumber data masuk ke slot, diberi source code, diproses, lalu berubah menjadi normalized dataset, visual result, AI interpretation, dan saved history.

## Palette

- Lab White `#F4F6F4`
- Graphite `#262A2D`
- Spectrum Blue `#315CFF`
- Sample Teal `#27A8A2`
- Reading Violet `#8056D9`
- Warning Orange `#F2783D`

## Typography

- Display: Urbanist Black
- Brand/body: Outfit
- Instrument labels: IBM Plex Mono

## Material Grammar

- sample cartridges;
- instrument slots;
- source labels;
- calibration bars;
- normalization channels;
- output trays;
- spectral bands.

## Scene Types

1. Source Intake
2. Parser Selection
3. Schema Calibration
4. Normalization Chamber
5. Metric Reading
6. AI Interpretation
7. History Shelf
8. Expandable Lab

## Narrative Sequence

`SELECT → INSERT → IDENTIFY → NORMALIZE → ANALYZE → INTERPRET → STORE → EXPAND`

## Avoid

- Medical laboratory stock photos.
- Test tube icons.
- Source logos memenuhi slide.
- Generic integrations logo cloud.
- AI diposisikan sebagai parser untuk semuanya.

## QA

- [ ] Setiap source punya cartridge code berbeda.
- [ ] Normalization terlihat sebagai proses.
- [ ] Deterministic analytics dan AI interpretation dipisahkan.
- [ ] History menjadi bagian narasi.
- [ ] Future source divisualkan sebagai slot, bukan klaim live.
